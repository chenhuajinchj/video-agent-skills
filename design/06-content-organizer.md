# 模块 6: video-content-organizer

## 功能概述

使用 NotebookLM 整理素材，生成结构化的内容大纲。

## 触发条件

- 「整理 NotebookLM 内容」
- 「生成视频大纲」
- 「Notebook 已准备好，URL 是 [链接]」

## 目录结构

```
video-content-organizer/
├── SKILL.md
└── references/
    └── outline-template.md
```

## 前置条件

- 用户已手动将素材上传到 NotebookLM
- 用户提供 Notebook URL

## 输入

- NotebookLM URL

## 输出文件

### outline.md

```markdown
# 视频大纲：[主题]

## 核心主题
[一句话概括视频核心观点]

## 预估信息
- 预估时长：X 分钟
- 预估字数：XXXX 字
- 论点数量：X 个

## 论点结构

### 论点 1：[标题]
- 核心观点：[观点描述]
- 案例：[案例描述]
- 出处：[来源信息]

### 论点 2：[标题]
- 核心观点：[观点描述]
- 案例：[案例描述]
- 出处：[来源信息]

### 论点 3：[标题]
- 核心观点：[观点描述]
- 案例：[案例描述]
- 出处：[来源信息]
```

## 处理流程

1. **连接 NotebookLM**
   - 使用 notebooklm-skill 访问用户的 Notebook
   - 获取已上传的素材内容

2. **提取关键信息**
   - 识别核心主题
   - 提取支撑论点
   - 找出案例和出处

3. **生成大纲**
   - 按模板格式组织内容
   - 确保每个论点有案例背书
   - 标注所有出处

4. **输出并等待确认**
   - 展示大纲预览
   - 等待用户确认或修改

## 大纲质量标准

| 要素 | 要求 |
|------|------|
| 核心主题 | 一句话，清晰明确 |
| 论点数量 | 3-5 个为宜 |
| 案例 | 每个论点必须有案例 |
| 出处 | 所有案例标注来源 |
| 时长预估 | 基于字数估算 |

## 依赖的外部 Skill

| Skill | 用途 |
|-------|------|
| notebooklm-skill | 访问 NotebookLM 内容 |

## 检查点

输出大纲后暂停，等待用户确认：
- 论点结构是否合理
- 案例是否准确
- 是否需要调整顺序或增删论点

## 使用示例

```
用户：Notebook 已准备好，URL 是 https://notebooklm.google.com/...
→ 连接 NotebookLM
→ 分析素材内容
→ 生成 outline.md
→ 展示预览，等待确认
```
