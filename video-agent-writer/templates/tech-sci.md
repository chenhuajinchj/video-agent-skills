# 技术科普模板（tech-sci）

适用主题：技术趋势分析、工具/产品对比、行业变革解读、技术原理科普、开源 vs 闭源等

## 开头模式表

| 主题类型 | 推荐模式 | 示例 |
|----------|----------|------|
| 行业趋势 | 反常识/数据冲击 | "2026 年，80% 的创业公司选择了开源模型，成本只有闭源的七分之一。" |
| 工具/产品对比 | 问题 | "都说开源模型够用了，那为什么大厂还在烧钱做闭源？" |
| 技术原理 | 故事/案例 | "上个月，一个三人团队用开源模型做了个产品，日活过万……" |

## 结构模板

递进式论证（现象 → 原因 → 影响 → 应对）：

```
开头（数据冲击 / 反常识 + 抛出核心问题）
  │
  ├── 第一层：现象是什么（数据、案例铺开）
  ├── 第二层：为什么会这样（技术原因、商业逻辑）
  ├── 第三层：这意味着什么（对行业、对个人的影响）
  └── 第四层：怎么应对（实操建议、选择框架）
  │
结尾（回扣核心问题 + 给出判断 + 行动建议）
```

也可用对比式结构（适合 A vs B 类主题）：

```
开头（抛出对比 + 核心问题）
  │
  ├── A 的优势和适用场景
  ├── B 的优势和适用场景
  ├── 关键差异对比（用数据说话）
  └── 不同场景下的最优选择
  │
结尾（给出判断框架 + 行动建议）
```

## 案例组合

推荐的论证方式：
- 数据对比 + 行业案例 + 技术原理 + 实操建议
- 大厂案例 + 创业公司案例 + 开发者视角
- 国际对比 + 国内现状 + 趋势预测

每个论证层按以下结构展开：
1. **论点**：一句话说清这一层要讲什么
2. **数据/案例**：具体数字、公司名、产品名、时间线
3. **出处标注**：独立一行
4. **小结**：一句话收束，衔接下一层

## 过渡句示例

- "数据摆在这了，但为什么会这样？"
- "光看表面不够，我们往深了挖一层。"
- "这对你意味着什么？"
- "说完了现状，来看看怎么选。"
- "有人会说……但实际上……"

## 结尾模式

结尾三要素：
1. 回扣开头的核心问题，给出明确判断（不骑墙）
2. 一个简洁的选择框架（帮观众做决策）
3. 一个具体的行动建议（观众看完马上能做的一件事）
